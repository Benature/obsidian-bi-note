# Bi-Note

[English](./README.md) | [中文](./README.zh-CN.md)

`Bi-Note` 是一个 Obsidian 插件，用来在日历中可视化多个文件夹里的笔记。

它不局限于单一的 Daily Note 目录。你可以定义多个笔记来源，把它们组合成不同的日历视图，并一眼看出每天哪些笔记已经存在。点击某个来源格子时，如果笔记已存在就直接打开；如果不存在，就可以创建并打开。

## 功能特性

- 支持多个日历视图，适配不同工作流
- 每个视图可配置多个笔记来源
- 使用颜色区分不同来源
- 支持跨多个文件夹管理笔记
- 点击即可打开已有笔记或创建缺失笔记
- 支持使用类 moment 的日期占位符配置路径模板
- 可调节来源格高度和日期格最小高度
- 支持中英文界面

## 工作方式

每个日历视图都包含一个或多个笔记来源。

每个来源包含：

- 名称
- 颜色
- 路径模板

对于日历中的每一天，Bi-Note 都会检查每个来源对应的笔记是否存在。如果存在，来源指示格会显示为配置的颜色；如果不存在，则保持空白。点击该格子即可打开或创建对应笔记。

## 使用场景示例

- 将个人日志和工作日志分别存放在不同文件夹中统一查看
- 在一个日历里对比多个日记或记录来源
- 为不同项目或不同时间维度建立独立日历视图
- 在不改变现有文件夹结构的前提下并行管理多套笔记体系

## 路径模板

路径模板支持常见的类 moment 日期占位符，例如 `YYYY`、`MM`、`DD`、`ddd`、`dddd`。

示例：

- `daily/YYYY/MM/YYYY-MM-DD`
- `work-log/YYYY/YYYY-MM-DD`
- `journal/YYYY/MM/YYYY-MM-DD_ddd`

普通路径文本会原样保留；如果路径末尾没有 `.md`，Bi-Note 会自动补上。

## 使用方法

1. 打开 **设置 -> 第三方插件 -> Bi-Note**。
2. 创建或编辑一个日历视图。
3. 为该视图添加一个或多个笔记来源。
4. 为每个来源设置路径模板。
5. 从设置页、命令面板或左侧功能区图标打开该视图。
6. 在日历中点击任意来源指示格，即可打开或创建目标笔记。

## 设置项

Bi-Note 当前支持：

- 插件界面语言：跟随系统、中文、英文
- 创建缺失笔记前是否确认
- 全局来源格高度
- 全局日期格最小高度
- 多个日历视图
- 每个日历视图中的多个笔记来源

## 命令

Bi-Note 会为每个日历视图注册一条命令：

- `打开日历：<视图名称>`

这样可以直接通过命令面板打开指定视图。

## 开发安装

```bash
npm install
```

启动监听构建：

```bash
npm run dev
```

执行生产构建：

```bash
npm run build
```

运行 lint 检查：

```bash
npm run lint
```

## 手动安装

将发布文件复制到：

```text
<your-vault>/.obsidian/plugins/bi-note/
```

必需文件：

- `main.js`
- `manifest.json`
- `styles.css`

然后重新加载 Obsidian，并在 **设置 -> 第三方插件** 中启用 **Bi-Note**。

## 发布说明

发布新版本时：

1. 更新 `manifest.json`。
2. 更新 `versions.json`。
3. 创建一个 GitHub Release，标签名必须与插件版本完全一致，且不要带 `v` 前缀。
4. 上传 `main.js`、`manifest.json` 和 `styles.css` 作为发布附件。
